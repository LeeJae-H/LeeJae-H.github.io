## Cloud Computing
- **클라우드 컴퓨팅**
    - 클라우드 컴퓨팅은 일반적으로 인터넷 기반 컴퓨팅의 한 형태로, 개인이나 기업이 자신의 컴퓨터가 아닌 클라우드에 연결된 다른 컴퓨터로부터 컴퓨팅 리소스를 활용하는 기술을 의미한다. 
    - 사용자는 물리적 데이터 센터와 서버를 직접 구입하고 관리하는 대신, AWS와 같은 클라우드 공급자를 통해 컴퓨팅 파워, 애플리케이션, 스토리지, 데이터베이스 등 다양한 기술 서비스에 필요한 만큼 접근할 수 있다. 즉, 사용자가 직접 관리하지 않고도 컴퓨터 시스템 리소스를 필요할 때 즉시 활용할 수 있다. 사용자는 실제 사용량에 따라 비용을 지불한다. 
    - AWS와 같은 클라우드 공급자는 컴퓨터 시스템 리소스(CPU, 디스크, 서버, 스토리지, 네트워크 등)를 인터넷을 통해 필요 시 바로 제공(on-demand availability)한다. 
    - 클라우드 컴퓨팅은 리소스의 온디멘드 제공과 분산 처리를 통해 유연하고 탄력적인 컴퓨팅 환경을 제공하며, 이는 기업들이 변동하는 수요에 신속하게 대응하고 비용을 최적화하는 데 도움을 준다.

- 클라우드 컴퓨팅의 사용자
    - 모든 유형, 규모 및 산업의 조직이 데이터 백업, 재해 복구, 이메일, 가상 데스크톱, 소프트웨어 개발 및 테스트, 빅데이터 분석, 고객 대상 웹 애플리케이션 등 다양한 사용 사례에 클라우드를 활용하고 있다.

## 가상화와 클라우드 컴퓨팅
<center><img src="https://github.com/LeeJae-H/LeeJae-H.github.io/assets/122717063/9ac74dc3-29d9-4662-a385-6be53f251021" width="500" height="220"></center>
- 클라우드 컴퓨팅을 실현하기 위해 여러 기술이 사용되는데, 그 중 핵심 기술 중 하나가 가상화이다.
    - 가상화 없이도 클라우드 컴퓨팅은 가능하지만, 클라우드 컴퓨팅을 더 효율적으로 구현하고 관리하기 위해 여러 기술들을 사용하며, 가상화는 그 기술들 중 하나이다.
    - 가상화는 기술, 클라우드는 그 기술을 이용하여 제공되는 환경이라고 볼 수 있다.

- 클라우드 공급자는 자체 데이터 센터를 관리하고, 하드웨어 리소스를 사용하는 서로 다른 가상 환경을 생성, 관리, 제공하는 역할을 한다. 
    - 가상화를 통해 단일 물리적인 서버의 리소스를 가상 서버로 분할하고 이를 클라우드 환경에서 관리하면, 리소스를 더욱 효율적으로 활용하고 필요에 따라 빠르게 확장할 수 있는 장점을 얻는다.
    - 서버1[(OS<APP), (OS<APP), (OS<APP)], 서버2[(OS<APP), (OS<APP)]
    - (OS<APP)은 가상화된 환경이고, 이것을 제공하는 것이다.
    - AWS 클라우드 환경에서, 리소스를 작은 단위로 빠르게 제공하는 데에 가상화가 중요한 역할을 한다. 
    - 예를 들어, AWS EC2는 컴퓨팅 파워를 빠르게 제공하는데, 이것의 원동력은 가상화이다. AWS는 이미 가동 중인 있는 서버를 효과적으로 가상화하여 컴퓨팅 파워를 제공한다. 즉, 이미 구축된 가상화 가능한 서버의 한 부분을 할당해주는 것이다. 
    - 여기서 컴퓨팅 파워는 인스턴스의 CPU, 메모리, 스토리지, 네트워크 처리 능력 등과 관련된 개념으로, 이를 통해 인스턴스가 작업을 수행하고 계산을 처리하는 능력을 의미한다.

## Cloud Service
- **클라우드 컴퓨팅 모델(서비스 제공 형태)** : 클라우드에서 무엇을 제공하느냐에 따라
<center><img src="https://github.com/LeeJae-H/LeeJae-H.github.io/assets/122717063/20a766f5-dae3-446f-beab-b4bf21678a51" width="500" height="300"></center>

1. **IaaS(Infrastructure as a Service)**  
    - IaaS는 클라우드 IT를 위한 기본 빌딩 블록을 제공한다. 주로 네트워킹, 컴퓨팅(가상 또는 전용 하드웨어) 및 데이터 스토리지 액세스를 제공한다. 기존 온프레미스 리소스와 유사한 유연성과 관리 제어 기능을 제공한다.
    - 인프라(Computing, Storage, Network) 제공
    - 사용자는 OS를 직접 설치하고 필요한 소프트웨어(APP)를 직접 개발해서 사용
    - 가상의 컴퓨터를 임대하는 것과 유사한 개념
    - ex) AWS EC2, AWS S3, Azure VM 등  

2. **PaaS(Platform as a Service)**
    - PaaS를 사용하면 기본 인프라(일반적으로 하드웨어와 OS)를 관리할 필요가 없어 애플리케이션 개발과 관리에 집중할 수 있다. 애플리케이션 실행과 관련된 리소스 구매, 용량 계획, 소프트웨어 유지 관리, 패치 작업 또는 다른 모든 획일적인 작업에 대한 부담 없이 더욱 효율적으로 운영할 수 있다.
    - 인프라 + (OS + 런타임 환경(앱 실행에 필요한 구성 요소)) 제공
    - 바로 코드만 올려서 돌릴 수 있도록 구성(짜여진 코드를 여기다가 넣기만 하면 이 애플리케이션이 돌아가서 우리들이 원하는 서비스를 구성할 수 있다)
    - ex) Firebase, Google App Engine, Heroku 등

3. **SaaS(Software as a Service)**
    - SaaS는 서비스 공급자에 의해 실행되고 관리되는 완전한 제품을 제공한다. 대부분의 경우 SaaS라고 하면 웹 기반 이메일과 같은 최종 사용자 애플리케이션을 말한다. . SaaS를 사용하면 유지 관리, 인프라 구성 등을 고려할 필요 없이 소프트웨어 사용 방법에만 집중하면 된다.
    - 인프라 + (OS + 런타임 환경) + APP(필요한 소프트웨어) 제공
    - 사용자는 다른 세팅 없이 서비스만 이용
    - 완전한 서비스 자체를 제공하는 것
    - ex) Gmail, DropBox, Slack, Google Docs 등

- **클라우드 컴퓨팅 배포 모델(서비스 유형)** : 클라우드 컴퓨팅을 어떻게 제공하느냐에 따라

1. **Public Cloud**
    - 인터넷에 접속 가능한 모든 사용자를 대상으로 하는 클라우드 서비스 모델이다. 클라우드 서비스 제공자(CSP)가 하드웨어, 소프트웨어를 관리한다. 데이터나 기능, 서버 같은 자원은 각 서비스에서 사용자 별로 권한 관리가 되거나 격리되어, 서비스 사용자 간에는 전혀 간섭이 없다는 장점이 있다.
    - 공개형(클라우드)
    - 모든 부분이 클라우드에서 실행
    - 낮은 비용과 높은 확장성
    - ex) AWS, Microsoft Azure, Google Cloud 등
    
2. **Private Cloud**
    - 제한된 네트워크 상에서 특정 기업이나 특정 사용자만을 대상으로 하는 클라우드 서비스 모델이다. 기업이 자원의 제어권을 갖고 있으며, 서비스의 자원과 데이터는 기업 내부에 저장된다. 따라서 보안성이 매우 뛰어나며, 개별 고객의 상황에 맞게 클라우드 기능을 커스터마이징 할 수 있다는 장점이 있다.
    - 폐쇄형(온프레미스)
    - 높은 수준의 커스터마이징 가능하며, 보안이 중요한 기업에서 선호됨
    - 높은 초기 비용과 유지 보수 비용
    - ex) 기업이 자체적으로 구축한 클라우드 환경
    
3. **Hybrid Cloud**
    - Public Cloud와 Private Cloud를 병행하여 사용하는 방식이나 Public Cloud(가상 서버)와 Private Cloud(물리 서버)를 결합한 형태를 의미한다. 이러한 클라우드 서비스 모델은 Public Cloud의 유연성, 경제성, 신속성과 Private Cloud(온프레미스)의 보안성, 안정성 등을 동시에 얻을 수 있다는 장점이 있다. 주요 데이터의 보안과 안정성을 Private Cloud(온프레미스)에서 유지하면서도 Public Cloud의 자원 확장 능력과 비용 효율성을 활용하여 비즈니스 목표를 달성하는 데 도움을 준다.
    - 최근 기업들은 모든 업무를 완전히 클라우드로 이전하기보다는, 주요 데이터를 온프레미스 환경에 유지하고, 예측하기 어려운 트래픽(이벤트 또는 신규 서비스 등의 특정한 상황)에서는 필요에 따라 Public Cloud를 활용하는 구성이 더 빈번해지고 있다.
    - 폐쇄형에서 공개형으로의 전환 과정에서 유용하게 활용되며, 또한 폐쇄형 환경의 백업 솔루션으로도 활용된다.

### Serverless Computing
- 서버리스 컴퓨팅은 클라우드 컴퓨팅 모델의 하나이다.
    - 클라우드 제공자가 서버의 프로비저닝, 운영 및 관리를 담당하므로 개발자는 코드 작성에 집중할 수 있다.
    - 즉, 서버리스는 서버가 없는 것이 아니고, 서버를 직접 관리할 필요가 없다는 의미이다.
    - 비용은 소비한 자원의 양에 기반을 둔다.

### Serverless Computing Model
- 서버리스의 대표적인 두 가지 구현 방식 : BaaS, FaaS
- **BaaS(Backend as a Service)**
    - ex) Firebase, AWS Amplify, Kinvey, Parse 등
    - 백엔드 서비스(데이터베이스, 스토리지, 인증, 푸시 알림 등)를 제공하는 클라우드 기반 서비스이다.
        - 개발자는 서버를 프로비저닝하고 관리할 필요 없다. 모든 것이 BaaS 제공자에 의해 처리된다.  


- **FaaS(Function as a Service)**
    - ex) AWS Lambda, Microsoft Azure Functions, Google Cloud Functions
    - 이벤트 기반의 작은 함수의 실행을 관리해주는 클라우드 기반 서비스이다.
        - 개발자가 코드를 실행하기 위한 서버를 프로비저닝하거나 관리하지 않아도 된다. 모든 것이 FaaS 제공자에 의해 처리된다.
    - 함수(코드)를 작성하고 업로드하기만 하면 클라우드 공급자가 이 함수(코드)를 실행하고 관리해준다.
        - 함수는 이벤트(HTTP 요청, 파일 업로드 등)가 발생할 때만 실행되며, 실행 후에는 리소스가 자동으로 해제된다.
    - 무분별하게 FaaS을 사용하게 되면 비용이 많이 부과된다. FaaS로 서버를 완전히 대체하는 것보다는, 필요에 따라 어떠한 기능만 대체하는 것이 효율적이다.
